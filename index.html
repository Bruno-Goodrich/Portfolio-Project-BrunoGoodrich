Filters
Github Repository

Overview

In this project I wast tasked with making filters for images. I did this by manipulating the red, green, blue, and alpha values of the pixels that make up the image. I created a sad filter that makes the image more blue and adds a vignette with code written by the one and only Tom Hinkle. I also created a color shift filter that kinda works and has options to change the R,G,B, and alpah added. There is also a brightness that was made as an introduction to how filters works.

Code Example

This is the code for the sad filter. It changes the image to be more blue with a vignette. The filter itself is an object with two methods. The first method is name which is, well, it's name and the second is the apply method which does the work of creating the filter. The way it works is by getting the array of pixels within the image and adding 50 to the red value of a pixel, 100 to the blue value, 50 to the green value, and subtracting 100 from the alpha value. It knows these values by using modulo of the 4 items of the pixel's array and saying if its the first item it is the red value, if it's the second it's green, third is blue, fourth is alpha. The last part calls upon another filter that adds a vignette and it's apply method to appear on the filtered image. Then it will be called and apply the filter.

Overview

In this project we had to make charts/graphs using car crash data from Massachusets. This was a group project so I worked with My pals Lucien and Matt. We chose to focus our data on bus crashes ad make a barchart of how many crashes happen in each town. I created the graph and helped create the list of towns in each county. I mainly did coding and debugging.

Purpose and Function

The purpose of this project was to to allow a user to be able to see the number of school bus accidents in Massachusetts. The function of the project was to make a chart that displays the number of of school bus road incidents in Massachusetts in each town. It sorts these towns in counties to allow easier searchin and a smaller, easier to read graph.

Code Example one

To put it simply, this function takes the list of town names and counts the crashes in each town. It then makes an array with the name of the town and the amount of crashes in it. More specifically the newPieCounts and newPieData create a array of objects that consist of the name of a town and the number crashes. The filteredData function is what does the counting. It looks through the data, only at the part of the rows labled as "City Town Name", and adds each name it finds as the name of the object in the array. When it finds a repeat of that name it adds one to the crashs number in the object. So once it finds a town like Lowell, it will add an object to the array with the name and a crash, once it finds Lowell in the csv data again, it will add one to the crash counter. It then puts these obects into the array at the start.
